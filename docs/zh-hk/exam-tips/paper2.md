## 各選修策略

### 卷 2A · 資料庫

- **參考表** 知道在哪。不背 SQL 關鍵字 —— 直接看參考。
- **寫 SQL 順序：**
  1. `FROM` / `JOIN`（涉及哪些表？）
  2. `WHERE`（過濾行）
  3. `GROUP BY`
  4. `HAVING`
  5. `SELECT`
  6. `ORDER BY`
  7. `LIMIT`
- **JOIN 時給表起別名**（`Student s INNER JOIN Score sc`）。
- 設計題旁**先畫 ER 圖**。
- 語句末尾加 `;`，顯得專業。

### 卷 2B · 網絡應用開發

- 明確區分**客户端 vs 服務器端**職責。
- 用户輸入**絕不信任**，客户端 + 服務器端**雙重校驗**。
- PHP 資料庫代碼用**預處理**。
- HTML 包含 `<!DOCTYPE html>` 和 `<meta charset="UTF-8">`。
- CSS 題展示至少一個選擇器、屬性、值。

### 卷 2C · 演算法與編程

- 題目未明示用 Python，**偽程式碼優先**。
- 複雜代碼加**註釋**。
- **跑一遍樣例**再宣告正確。
- 準備**比較演算法**（O 記號好用即使非必要）。

## 評分方式

評卷用「踩點」式：

- **一個概念 = 一分**。
- 即使最終答錯有**方法分**（如 JOIN 結構正確但出現筆誤）。
- 清晰加**額外分**（註釋、縮進、起名）。

> 不確定時，寫**更結構化、更清晰**的內容，未必更長。

## 讀題清單

1. 圈**命令動詞**（Write、Explain、Identify…）。
2. 圈**資料上下文**（哪張表？哪種語言？）。
3. 看**分值**。
4. 找**約束**（「用 JOIN」、「不用子查詢」、「只一個迴圈」）。

## 卷二專屬常見錯誤

- **2A：** 用 `WHERE COUNT(*) > 5`（應 `HAVING`）。
- **2A：** 把 `<>` 寫成 `!=`。
- **2B：** 題問 cookie 卻用了 localStorage。
- **2B：** 不用 `PDO::prepare()` → 可被 SQL 注入。
- **2C：** `for i in range(n)` 與 `range(1, n+1)` 弄錯。
