> 2 小時 · MC 22% + Section B 33% · 僅必修部分。

## 時間分配

| 部分 | 大致分數 | 建議時間 |
|------|---------|---------|
| Section A（MC） | 36 × 1 = 36 | 30–40 分鐘 |
| Section B（短題 + 結構題） | ~55 | 70–80 分鐘 |
| 檢查 | — | 10 分鐘 |

## Section A · 選擇題

### 每題流程

1. **遮住選項**，先在腦裏給出答案。
2. **打開選項** 選最接近的。
3. **不確定就排除** 至少 2 個錯的。
4. **果斷推進**，別一題糾纏 3 分鐘。
5. **標記不確定** 的，留時間回看。

### MC 常見陷阱

| 陷阱 | 例子 |
|------|------|
| 幾乎正確 | 「TCP 做什麼？」A) 路由包（那是 IP）B) 可靠傳輸（正確） |
| 否定式 | 「以下哪項**不是** normalisation 的好處？」漏看 NOT |
| 單位不符 | 「1 KB =」 1024 位元組（不是 1000） |
| 範圍混淆 | 題描述 LAN，但選項對比 WAN |

## Section B · 短題與結構題

### 結構題結構

通常 4–6 個子題串成一道：

```
Q4. 情境段：學校圖書館系統
   (a) 定義 X。 [2]
   (b) 識別 Y 的兩個屬性。 [2]
   (c) 寫 Z 的偽程式碼。 [4]
   (d) 評估設計相對 W 的優劣。 [4]
```

每個子題當作小論文。情境段只在引用時重讀。

### 答案長度按分數

| 分數 | 典型長度 |
|------|---------|
| 1 | 一短語或單一事實 |
| 2 | 兩事實 或 一事實 + 一例 |
| 3–4 | 含定義 + 例的結構段 |
| 5–6 | 多段 + 圖 + 比較 |

### 命令動詞認真對待

| 動詞 | 期望回答 |
|------|---------|
| **State / List** | 簡短事實 |
| **Describe** | 句級解釋 |
| **Explain** | 為什麼/如何，含因果 |
| **Compare** | 兩側表或段 |
| **Evaluate** | 優 + 劣 + 結論 |
| **Design** | 計劃 + 理由（如 ER 圖 + 解釋） |

### 答題模板

#### 「解釋為何 X 發生」

> X 發生因為 **\<原因\>**。這是因為 **\<機制\>**。結果是 **\<效果\>**。例如 **\<具體實例\>**。

#### 「比較 A 與 B」

> A 是 **\<簡述\>**，而 B 是 **\<簡述\>**。就 **\<標準 1\>** 而言，A **\<…\>**，B **\<…\>**。就 **\<標準 2\>** 而言，A **\<…\>**，B **\<…\>**。因此 A 更適合 **\<場景\>**。

#### 偽程式碼作答

務必 HKEAA 風格：

```text
INPUT n
total ← 0
FOR i ← 1 TO n
    total ← total + i
END FOR
OUTPUT total
```

用 `←` 不用 `=`。用 `END FOR`、`END IF` 收尾。
