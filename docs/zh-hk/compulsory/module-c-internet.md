## a. 網絡與互聯網基礎（9h）

### LAN vs WAN

| 特性 | LAN | WAN |
|------|-----|-----|
| 覆蓋 | 單個建築/校園 | 城市、國家、全球 |
| 擁有者 | 通常單一機構 | 多方 |
| 速度 | 較高（1 Gbps+） | 單用户較低 |
| 例子 | 學校機房 | 互聯網本身 |

### IPv4 vs IPv6

| 特性 | IPv4 | IPv6 |
|------|------|------|
| 長度 | 32 位 | 128 位 |
| 表示 | 點分十進制：`192.168.1.10` | 冒號十六進制：`2001:db8::1` |
| 地址空間 | ~43 億 | ~3.4 × 10³⁸ |
| 為何需要 IPv6 | IPv4 地址用盡 | 面向未來、路由簡化 |

### 網絡硬件

| 裝置 | 功能 |
|------|------|
| **NIC** | 把電腦接入網絡 |
| **交換機** | 用 MAC 地址在 LAN 內轉發幀 |
| **路由器** | 用 IP 地址在網絡間轉發資料包 |
| **調製解調器** | ADSL/電纜/光纖的調製解調 |
| **無線接入點** | 讓無線客户端接入有線 LAN |

### 傳輸介質

- **UTP**（非屏蔽雙絞線）—— 銅線，辦公室常見
- **同軸電纜** —— 舊式、有線電視
- **光纖** —— 光在玻璃中，速度最快、距離最遠
- **微波 / 衞星** —— 長距離無線
- **藍牙 / Wi-Fi** —— 短距離無線

### 互聯網接入比較

| 方式 | 速度 | 價格 | 移動性 | 備註 |
|------|------|------|--------|------|
| 撥號 | ~56 kbps | 便宜 | 無 | 已淘汰 |
| ADSL | 最高 ~24 Mbps | 低 | 無 | 用電話線 |
| 有線 | 最高 ~1 Gbps | 中 | 無 | 用電視線 |
| 光纖入户 | 1–10 Gbps | 中 | 無 | 香港主流 |
| 4G LTE | ~100 Mbps | 中 | 高 | 移動 |
| 5G | 1–10 Gbps | 高 | 高 | 最新移動 |
| 衞星 | 不定 | 高 | 高 | 偏遠地區 |

### TCP/IP 基礎

- **TCP** 提供可靠、順序、錯誤檢查的傳輸（按**端口號**區分服務）。
- **IP** 負責**尋址**與**路由**。
- 二者合作實現端到端通信。

## b. 互聯網服務與應用（5h）

### URL 結構

```
https:// www.example.com :443 /path/to/page ?id=123 #section
└──┬──┘ └──────┬───────┘ └┬─┘ └────┬─────┘ └──┬──┘ └──┬───┘
 scheme    hostname     port      path     query  fragment
```

### DNS 示意

```
用户輸入  www.example.com
       │
       ▼
   ┌────────────┐    ┌────────────┐
   │  瀏覽器    │ ─▶ │ DNS 伺服器 │  解析 www.example.com
   └────────────┘    └────────────┘
        ▲                  │
        │ IP 93.184.216.34 │
        └──────────────────┘
        │
        ▼
   向 93.184.216.34 發 HTTPS 請求
```

### HTTP vs HTTPS

| 特性 | HTTP | HTTPS |
|------|------|-------|
| 加密 | 無 | TLS/SSL |
| 默認端口 | 80 | 443 |
| 隱私 | 明文 | 加密 |
| 現今使用 | 很少 | 幾乎全部 |

### Web 文件格式

| 需要 | 格式 |
|------|------|
| 圖像 | jpg/jpeg、png、gif、webp、svg |
| 音頻 | mp3、ogg、wav |
| 影片 | mp4（h.264/h.265）、webm |
| 文件 | pdf |

### 互聯網服務

| 服務 | 協議 | 用途 |
|------|------|------|
| Web | HTTP/HTTPS | 瀏覽 |
| Email | SMTP、POP3、IMAP | 收發郵件 |
| FTP | FTP/SFTP | 文件傳輸 |
| 遠程登錄 | SSH | 伺服器管理 |
| 聊天 | XMPP、專有 | 即時通信 |
| 流媒體 | RTMP、HLS、DASH | 語音、視像會議、直播 |

## c. 網頁編寫基礎（3h）

### 最小 HTML 頁面

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>你好 HKDSE</title>
</head>
<body>
  <h1>你好，世界</h1>
  <p>這是我的第一個網頁。</p>
  <a href="https://www.hkeaa.edu.hk">HKEAA</a>
  <img src="logo.png" alt="logo">
</body>
</html>
```

不需要背誦所有標籤 —— 但要認得結構。

### 網頁組織原則

- **導航** 清晰
- **鏈接、表格、框架、多媒體** 放置得當
- 合理的**配色、背景、字體**
- 匹配**目標受眾**（小朋友 vs 公司 vs 學術）

## d. 互聯網的威脅與安全（14h）

### 威脅清單

| 威脅 | 做了什麼 |
|------|---------|
| 病毒 | 附着在其他程序上自我複製 |
| 蠕蟲 | 在網絡上獨立自我複製 |
| 木馬 | 偽裝成合法軟件，開後門 |
| 間諜軟件 | 未經同意追蹤用户 |
| 勒索軟件 | 加密資料索要贖金 |
| 未授權存取 | 無權限登入 |
| 竊聽 | 捕獲傳輸中的資料 |
| 通過動態網頁入侵 | SQL 注入、XSS |
| DoS / DDoS | 用流量淹沒服務 |

### 防禦措施

| 措施 | 阻止 |
|------|------|
| 瀏覽器安全設置 | 釣魚、悄悄下載 |
| 防病毒軟件 | 設備惡意軟件 |
| 認證（密碼、2FA、生物） | 未授權存取 |
| 存取權限控制 | 內部濫用 |
| 防火牆 | 不期望的進出流量 |
| WPA / WPA2 / WPA3 | 無線竊聽 |
| VPN | 加密用户與網絡間資料 |
| 定期 OS / 軟件更新 | 已知漏洞 |

### 加密基礎

| 概念 | 備註 |
|------|------|
| 對稱加密 | 一把共享密鑰（如 AES） |
| 非對稱加密 | 公開金鑰/私人金鑰對（如 RSA） |
| 香港 PKI | 香港郵政運營的公開金鑰基礎設施 |
| 密鑰越大 | 越安全但越慢 |

### 認證與授權

- **認證** 回答「你是誰？」—— 密碼、令牌、生物、數碼證書。
- **授權** 回答「你能做什麼？」—— 存取控制列表、角色。

### 在線交易安全

- **SSL/TLS** 把 HTTP 加密成 HTTPS。
- **智能卡、安全令牌** —— 硬件認證。
- **手機 SMS 一次性密碼** —— 香港銀行常用。

## 常見錯誤

- 把**互聯網**稱為 LAN（互聯網是 WAN 的 WAN）。
- 混淆 **交換機**（只在 LAN）與 **路由器**（跨網絡）。
- 以為 HTTPS 「完全安全」—— 它只保護傳輸。
