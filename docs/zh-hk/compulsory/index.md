# 卷一 · 必修部分總覽

> **比重：** 全科 55% · **時長：** 2 小時 · **形式：** Section A（MC，22%）+ Section B（短/結構題，33%）

必修部分在 **卷一（Paper 1）** 考核。不論你的選修組合如何，每位考生都要做同一份卷一。因此它是 ICT 科目中分值最大的單一部分，也是你複習的首要目標。

## 五大模組速覽

| # | 模組 | 課時 | 為什麼重要 |
|---|------|------|----------|
| **A** | [資訊處理](./module-a-information-processing) | 37 | 卷二 2A 的基礎。培養資料思維與 Excel / DBMS 技能。 |
| **B** | [電腦系統基礎](./module-b-computer-system) | 20 | 硬件 / 軟件能力。MC 題的穩定來源。 |
| **C** | [互聯網及其應用](./module-c-internet) | 31 | 網絡 + 安全。卷二 2B 的基礎。 |
| **D** | [計算思維與編程](./module-d-programming) | 48 | 最大的模組。支撐卷二 2C 與卷一的追蹤表題。 |
| **E** | [社會影響](./module-e-social) | 8 | 模組小、得分密度高。討論 + 倫理。 |

### 考核範圍

模組 A–E 的所有學習要點都可能出現。HKEAA 經常在一道結構題裏跨模組出題 —— 同一長題的不同子題可能從二進制（模組 A）跳到網絡協定（模組 C），最後落到偽程式碼追蹤（模組 D）。

### 卷一的評核目標

你需要展示：

1. **知識與理解** 電腦系統、資料、硬件、軟件、互聯網。
2. **應用** 把陳述性知識用到具體場景（如選擇合適的驗證方法）。
3. **分析** 比較兩種解決方案、追蹤偽程式碼、調試。
4. **綜合** 用多個概念設計新的解決方案。

課程指引中的行為動詞對應不同的認知層次：

| 動詞類 | 認知層次 |
|--------|--------|
| state, list, define, recognise | 回憶 |
| describe, explain, identify, demonstrate, apply | 應用 |
| distinguish, analyse, compare, evaluate | 分析 |
| develop, plan, design, construct | 綜合 |

題目以「**設計** 一個函式 …」開頭，意味着要寫完整的實現邏輯。以「**陳述** 兩者的差別」開頭，一句話即可。

## 必修部分學習建議

1. **順序很重要。** 大部分老師按 A → C → B → E → D 教學，讓前面的概念支撐後面的學習。自學者也建議如此。
2. **把模組 D 當成編程肌肉。** 編程不能臨時抱佛腳。從中四起每週至少跑一個 Python（或 C++）程序。
3. **用間隔記憶背詞彙。** 模組 B、C、E 詞彙量大。閃卡比重讀筆記更有效。
4. **與選修對照學習。** 模組 A 對應卷二 2A，模組 C 對應 2B，模組 D 對應 2C —— 一起溫習省時間。

::: tip 在瀏覽器裏練習 SQL
卷一 Section B 經常出 SQL `SELECT` 的輸出預測題。在 **[SQL Books](https://sqlbooks.codenav.dev)** 上交互式跑這些例子 —— 免費的瀏覽器 SQL 工作台，每個賬號自帶 MySQL 沙盒，無需配置、不怕弄壞。
:::
