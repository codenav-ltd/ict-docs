# 4.3 · 加密基礎

> **目標：** 解釋對稱 vs 非對稱加密、密鑰長度的作用，以及香港 PKI。

## 加密做什麼

**加密**用一個**密鑰**把可讀資料（**明文**）變成不可讀資料（**密文**）。只有有對的密鑰才能**解密**。

```
明文  ── [用密鑰加密] ──▶  密文
密文  ── [用密鑰解密] ──▶  明文
```

## 對稱加密

**一個密鑰**同時加解密。收發雙方須事先共享相同密鑰。

| 性質 | 詳情 |
|----------|--------|
| 速度 | 非常快 |
| 例子 | AES、DES（舊）、3DES |
| 問題 | 如何安全共享密鑰？ |

## 非對稱（公鑰）加密

數學上相關的兩把鑰匙：

- **公鑰** —— 與所有人分享。
- **私鑰** —— 保密。

```
用公鑰加密  ── 只有匹配的私鑰能解。
用私鑰簽名  ── 任何人都能用公鑰驗證。
```

| 性質 | 詳情 |
|----------|--------|
| 速度 | 慢（遠慢於對稱） |
| 例子 | RSA、ECC |
| 解決 | 密鑰分發問題 |

## 混合方式（HTTPS 所用）

1. 瀏覽器與服務器用**非對稱**加密安全交換**對稱會話密鑰**。
2. 然後切到**對稱**加密做實際資料 —— 快。

兩者之長：安全密鑰交換 + 快速大批加密。

## 密鑰長度重要

| 長度 | 對稱例子 | 非對稱例子 | 強度 |
|----------|-------------------|--------------------|----------|
| 56 位 | DES | — | 已破 |
| 128 位 | AES-128 | — | 強 |
| 256 位 | AES-256 | — | 很強 |
| 2048 位 | — | RSA-2048 | 當代強 |
| 3072+ 位 | — | RSA-3072+ | 面向未來 |

密鑰越大 → 越難暴力 → 也越慢。要權衡。

## 香港 PKI

香港的**公鑰基礎設施** (PKI) 由香港郵政經 e-Cert 服務運營：

- 向個人與機構簽發**數字證書**。
- 用於法律文件的**數字簽名**。
- 受《電子交易條例》承認。

## 數字簽名

數字簽名用**發送方私鑰**給消息哈希簽名。接收方用發送方**公鑰**驗證。

它證明：

- **真實性** —— 消息來自私鑰持有人。
- **完整性** —— 消息未被改。
- **不可抵賴** —— 發送方事後不能否認簽過。

## 考試式題目

> **題（5 分）：** 按密鑰、速度與密鑰分發問題比較對稱與非對稱加密。解釋 HTTPS 如何兩者並用。

**參考答案：**

- **密鑰**：對稱加密用**一把共享密鑰**做加解密；非對稱用**一對** —— 公鑰（廣泛分享）與私鑰（保密）。
- **速度**：對稱演算法 (AES) 計算快；非對稱 (RSA) 慢得多。
- **密鑰分發**：對稱需要事先用安全通道共享密鑰；非對稱允許公鑰公開傳遞解決了此問題。
- **HTTPS** 兩者合用：瀏覽器用服務器**公鑰**（來自證書）安全送出新生成的**對稱會話密鑰**；後續流量用對稱加密以求性能，只有那一次密鑰交換用慢的非對稱演算法。

## 關鍵要點

- 對稱 = 一密鑰、快。
- 非對稱 = 兩密鑰、慢、解決密鑰分發。
- 密鑰越大 = 加密越強。
- HTTPS 用混合方案。
