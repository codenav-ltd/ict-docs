## a. 基本機器組織（14h）

### 學習要點

- 解釋硬件功能：**輸入/輸出裝置、處理單元、總線、儲存裝置**。
- 解釋 **CPU** 結構與組成部分。
- 説明 **取指 - 譯碼 - 執行** 週期及暫存器、總線、ALU 的角色。
- 描述 **RAM、ROM、Cache** —— 以及內存大小、地址、字長與性能的關係。
- 描述輸入輸出裝置並按情境選用。
- 描述儲存裝置：存取方式、易失性、傳輸速率、容量。
- 説明處理器、內存、二級儲存、資料通信的**最新發展**。

### CPU 組件

```
          ┌────────────── CPU ───────────────┐
 輸入  ▶ │  ┌─────────┐  ┌────┐  ┌────────┐ │ ▶ 輸出
         │  │ 控制單元│  │ALU │  │暫存器  │ │
         │  └─────────┘  └────┘  │PC IR等 │ │
         │       ▲ 總線系統 ▲     └────────┘ │
         └───────│─────────│─────────────────┘
                 ▼         ▼
          ┌──────────┐   ┌────────────┐
          │  內存    │   │ 儲存裝置  │
          │ (RAM/ROM)│   │ (HDD/SSD)  │
          └──────────┘   └────────────┘
```

### 取指 - 譯碼 - 執行週期

1. **取指（Fetch）** —— PC 告訴 MAR 要讀取的地址，資料返回 MDR。
2. **譯碼（Decode）** —— 控制單元解碼 IR 中的指令。
3. **執行（Execute）** —— ALU 執行算術/邏輯；結果放入暫存器或內存。
4. **遞增** PC，重複。

### CPU 速度單位

| 單位 | 時間 | 用途 |
|------|------|------|
| 1 毫秒（ms） | 10⁻³ 秒 | 磁盤尋道時間 |
| 1 微秒（μs） | 10⁻⁶ 秒 | 舊 RAM 週期 |
| 1 奈秒（ns） | 10⁻⁹ 秒 | 現代 RAM 存取 |
| 1 皮秒（ps） | 10⁻¹² 秒 | CPU 指令時間 |

### 內存類型

| 類型 | 易失？ | 速度 | 價格 | 用途 |
|------|--------|------|------|------|
| **Cache** | 是 | 最快 | 最貴 | 緊貼 CPU 的活躍資料 |
| **RAM（DRAM）** | 是 | 快 | 中等 | 程序運行時的主內存 |
| **ROM** | 否 | 快（只讀） | 中等 | 固件（如 BIOS） |
| **二級儲存**（HDD/SSD/Flash） | 否 | 最慢 | 便宜 | 長期儲存 |

### 容量單位前綴

```
1 KB = 1024 位元組（不是 1000！）
1 MB = 1024 KB
1 GB = 1024 MB
1 TB = 1024 GB
```

SI 用 10 的冪，所以廠商宣傳的 1 TB（10¹² 位元組）比真正的 1 TB（2⁴⁰ 位元組）**小**。你應能解釋為什麼 Windows 顯示的容量比包裝少。

### 輸入輸出裝置

| 裝置 | 類型 | 用途 |
|------|------|------|
| 鍵盤、滑鼠、掃描儀、麥克風、攝影機、傳感器 | 輸入 | 捕獲用户動作或環境 |
| 顯示器、揚聲器、列印機、繪圖儀、投影儀 | 輸出 | 顯示或產出結果 |
| 觸摸屏 | 兼輸入輸出 | 移動 UI |

### 儲存裝置

| 裝置 | 存取 | 易失？ | 大致傳輸速率 | 容量 |
|------|------|--------|--------------|------|
| 磁盤（HDD） | 隨機 | 否 | ~100 MB/s | 最高 ~20 TB |
| 光碟（CD/DVD/Blu-ray） | 隨機 | 否 | ~50 MB/s | 最高 ~100 GB |
| 閃存（SSD / USB） | 隨機 | 否 | ~500 MB/s (SATA), ~7 GB/s (NVMe) | 最高 ~8 TB |
| 磁帶 | 順序 | 否 | ~400 MB/s | ~20 TB+ |
| 網絡儲存（NAS / 雲） | 網絡隨機 | 否 | 取決於網絡 | 幾乎無限 |

趨勢：體積更小、容量更大、速率更快。

## b. 系統軟件（6h）

### 學習要點

- 瞭解**系統軟件與應用軟件**的功能與硬件、用户之間的關係。
- 概述**作業系統**的基本功能及常見 OS。
- 説明**實用程式**與**驅動程式**的功能與必要性。
- 區分各種**運行模式**（批處理、實時、並行、分佈式、虛擬化）。

### 軟件層次

```
┌────────────────────────────┐
│   應用軟件                  │  Word、Chrome、Photoshop…
├────────────────────────────┤
│   系統軟件                  │  OS + 實用程式 + 驅動
├────────────────────────────┤
│   硬件                      │  CPU、RAM、裝置…
└────────────────────────────┘
       ▲
       │ 用户主要通過應用軟件交互。
```

### 作業系統功能

- **程序管理** —— 調度與切換。
- **內存管理** —— 分配 RAM、虛擬內存分頁。
- **文件管理** —— 目錄、權限。
- **設備管理** —— 驅動與 I/O。
- **安全與用户管理** —— 登錄、權限。
- **用户界面** —— GUI 或 CLI。

### 常見 OS

| OS | 典型用途 |
|----|---------|
| Windows | 桌面、辦公 |
| macOS | 創意專業 |
| Linux（Ubuntu、Debian、Fedora） | 伺服器、開發者 |
| iOS、Android | 移動設備 |
| Chrome OS | 輕量筆記本 |

### 實用程式

| 實用程式 | 功能 |
|---------|------|
| 資料壓縮 | 減小文件體積（zip、rar、7z） |
| 病毒檢測 | 檢測並清除惡意軟件 |
| 文件管理器 | 瀏覽、複製、移動、重命名 |
| 磁碟重組 | 重組 HDD 上的資料以加速存取 |
| 系統監控 | 顯示 CPU/RAM/磁盤使用 |

### 驅動程式

**驅動程式**讓 OS 與特定硬件通信。沒有正確驅動，設備即使物理連接也無法使用。

### 運行模式

| 模式 | 含義 | 例子 |
|------|------|------|
| **批處理** | 任務排隊、無人交互運行 | 月薪發放 |
| **實時處理** | 需立即響應 | 飛控、網銀 |
| **並行處理** | 多處理器同時處理一個任務的不同部分 | 科學模擬 |
| **分佈式處理** | 工作分散在多台聯網電腦 | 雲服務 |
| **虛擬化** | 一台物理機運行多個虛擬機 | 雲伺服器、沙盒 |

## 常見錯誤

- 把 **cache**（易失）當成 **ROM**（非易失）。
- 混淆 **固件**（儲存在 ROM 的軟件）與作業系統。
- 把 Windows 既算系統軟件又算應用軟件（它是系統軟件）。
- 忘記 **虛擬化** 是一種運行模式，而不僅指 VirtualBox。
