# 2.2 · 流程圖

> **目標：** 用標準符號讀畫流程圖。

## 標準符號

| 符號 | 形狀 | 用途 |
|--------|-------|-----|
| **Terminator 終止** | 圓角矩形 / 橢圓 | Start、End |
| **Process 過程** | 矩形 | 動作（賦值、計算） |
| **Input / Output 輸入 / 輸出** | 平行四邊形 | 讀 / 寫 |
| **Decision 判斷** | 菱形 | 是 / 否問題 |
| **Flow line 流向** | 箭頭 | 執行方向 |
| **Connector 連接器** | 小圓 | 接到別處（避線交） |
| **Subroutine call 子程式調用** | 帶雙豎線的矩形 | 調用另一子程式 |

## 例 · BMI 類別流程圖

```
            ┌────────────┐
            │  Start     │
            └────┬───────┘
                 │
            ┌────▼────────────┐
            │ Input weight, h │
            └────┬────────────┘
                 │
            ┌────▼─────────┐
            │ bmi ← w/h²   │
            └────┬─────────┘
                 │
              ◇──┴──◇  YES   ┌──────────────┐
              ◇bmi<18.5◇──▶  │Output "Under"│
              ◇──┬──◇         └────┬─────────┘
                 │ NO               │
              ◇──┴──◇  YES   ┌──────────────┐
              ◇bmi<24◇──▶    │Output "Norm" │
              ◇──┬──◇         └────┬─────────┘
                 │ NO               │
              ┌──┴──────────────┐   │
              │Output "Over/Obese"│ │
              └────┬─────────────┘ │
                   └──────┬────────┘
                          ▼
                     ┌────────┐
                     │   End  │
                     └────────┘
```

（HKEAA 以其漂亮的印刷形式重現流程圖 —— 但符號集相同。）

## 好流程圖的規則

- 每個過程**一入一齣**。
- **流向線**不應交叉（用連接器）。
- **判斷菱形**恰好一個 YES、一個 NO 分支。
- 嵌套不過 3 層 —— 更深就分解為子流程圖。

## 何時用流程圖 vs 偽程式碼

| 用流程圖 | 用偽程式碼 |
|---------------------|---------------------|
| 觀眾偏好視覺 | 觀眾能讀程式碼風格 |
| 展示分支邏輯 | 展示詳細步驟序列 |
| SBA 設計文件 | 編碼實現筆記 |

## 考試式題目

> **題（5 分）：** 畫流程圖：讀學生分數，≥ 50 輸出「Pass」，否則「Fail」。

**參考答案（文字表示）：**

1. Start。
2. Input mark。
3. 判斷：mark ≥ 50？
   - YES → Output 「Pass」 → End。
   - NO → Output 「Fail」 → End。

考試時實際畫圖形 —— 終止、平行四邊形給輸入、菱形給判斷、平行四邊形給輸出、終止 End。

## 關鍵要點

- 五種符號幾乎涵蓋一切。
- 一入 / 一齣；不交叉。
- 流程圖配偽程式碼更清楚。
