# 模組 A · 資訊處理

## a. 資訊處理簡介（3h）

### 學習要點

- 識別並分析**信息系統**的組成。
- 區分各種**信息處理過程**。
- 説出**資料與信息**的區別，並識別文字、數碼、圖像、音頻、影片等資料類型。
- 定義**信息時代**，討論**信息素養**在知識型社會中的重要性。

### 概念

- **信息系統的組成：** 目的、資料、過程、技術、人員。
- **信息處理過程：** 收集、組織、分析、儲存、處理、傳輸、展示。
- **資料 vs 信息：** 資料是原始材料；信息是被賦予上下文與意義的資料。

### 常見題目句式

- *「列出資料與信息的兩點差異。」*
- *「找出上述情境中的輸入 / 處理 / 輸出組件。」*

## b. 資料組織與控制（4h）

### 學習要點

- 在層級化資料組織中識別**資料、記錄、字段、文件、資料庫**。
- 解釋記錄如何被**組織、儲存、檢索**。
- 陳述**直接存取**與**順序存取**的優缺點和應用。
- 討論**資料控制**的需要並描述錯誤檢測方法。

### 關鍵知識

| 術語 | 定義 |
|------|------|
| 字段（Field） | 最小的有意義資料單元（如學生的姓） |
| 記錄（Record） | 相關字段的集合（如一個學生的整行資料） |
| 文件（File） | 相關記錄的集合 |
| 資料庫（Database） | 相關文件的結構化集合 |
| 直接（隨機）存取 | 用地址直接跳到某條記錄。個別查詢快，硬碟常用。 |
| 順序存取 | 從頭按順序讀。磁帶成本低，但隨機讀慢。 |

### 錯誤偵測 vs 防止

| 技術 | 目的 |
|------|------|
| **驗證（Validation）** | 軟件自動檢查（範圍、必填、格式、校驗碼） |
| **核實（Verification）** | 人工檢查（重新輸入、目視檢查、雙人輸入） |
| **奇偶校驗（Parity Check）** | 位級別檢查（奇/偶校驗位）用於傳輸和儲存 |

::: tip 記憶口訣
「**V**alidation 由程式碼 **V**alidate，**V**erification 由人 **V**iew。」
:::

## c. 資料表示（10h）

### 學習要點

- 區分**模擬資料**與**數碼資料**。
- 在**十進制 / 二進制 / 十六進制**之間互轉。
- 進行簡單的**二進制加減**，分析**溢出**。
- 用**二進制補碼**表示負整數。
- 知道字符如何用 **ASCII、Big-5、GB、Unicode** 表示。
- 簡單瞭解**圖像、音頻、影片**的數碼化方式。
- 比較常見**文件格式**（bmp、png、jpg、wav、mp3、avi、mpeg4、txt、docx、odt、pdf）。

### 進制對照表

```text
十進制  二進制    十六進制
0       0000      0
1       0001      1
2       0010      2
...
9       1001      9
10      1010      A
11      1011      B
12      1100      C
13      1101      D
14      1110      E
15      1111      F
```

### 二進制補碼（8 位例子）

用 8 位表示 **−5**：

1. 寫出 **+5** 的二進制：`0000 0101`
2. 取反：                  `1111 1010`
3. 加 1：                  `1111 1011`

### 字符編碼

| 編碼 | 大小 | 用途 |
|------|------|------|
| ASCII | 7 位（儲存為 1 位元組） | 基礎英文 |
| Big-5 | 2 位元組 | 繁體中文 |
| GB | 2–4 位元組 | 簡體中文 |
| Unicode（UTF-8） | 1–4 位元組 | 全球所有文字 |

> 課程指引説**無需**背誦具體編碼，但要理解為什麼更大字符集需要更多位數。

### 文件格式比較

| 類型 | 常見格式 | 壓縮 | 適合 |
|------|----------|------|------|
| 圖像 | bmp、png、jpg | bmp 不壓縮，png 無損，jpg 有損 | jpg 用於照片，png 用於圖標/截圖 |
| 音頻 | wav、mp3 | wav 不壓縮，mp3 有損 | mp3 用於流媒體/分發 |
| 影片 | avi、mpeg4 | 視情況，mpeg4 有損 | mpeg4 用於網絡 |
| 文件 | txt、docx、odt、pdf | 視情況 | pdf 歸檔、txt 純文本 |

## d. 資料操控與分析（20h）

最大的子話題，考得最多。分為試算表與資料庫兩部分。

### 試算表技能

- **單元格引用：** 相對（`A1`）、絕對（`$A$1`）、混合（`$A1`、`A$1`）
- **函式：** SUM、AVERAGE、MIN、MAX、COUNT、COUNTA、COUNTIF、IF、IFS、VLOOKUP、HLOOKUP、INDEX、MATCH、ROUND、INT、MOD、ABS
- **數學、邏輯、關係運算符**
- 單條件或多條件的**篩選、查找、排序**
- **跨工作表**動態引用
- **樞紐分析表與透視圖**做聚合
- **What-if 分析** —— 單變量求解、方案管理器、模擬運算表

### 資料庫技能（DBMS）

- 應用資料組織概念創建並維護**簡單資料庫**。
- 創建並使用**表單**錄入資料。
- 通過**查詢**提取與操控資料，創建**報表**。
- 追蹤並理解簡單的 **SQL 語句**（不要求寫複雜 SQL，那是卷二 2A 的內容）。

::: tip 不必為讀懂 SELECT 就裝 MySQL

### 例子 · 試算表

> **題：** 工作表 `B2:B13` 儲存 12 個月銷售額。在 `C2` 寫一個公式：若 `B2` 大於年度平均值顯示「Above」，否則顯示「Below」。

```text
=IF(B2 > AVERAGE($B$2:$B$13), "Above", "Below")
```

注意複製公式時需用 `$` 鎖定範圍。

### 例子 · SQL 追蹤

```sql
SELECT name, score
FROM   students
WHERE  class = 'F4A'
ORDER  BY score DESC;
```

預期：返回 F4A 班所有學生的 `name` 和 `score`，按 `score` 降序。

## 常見錯誤

- 混淆 **validation** 與 **verification**（用上面的 V 口訣記憶）。
- 二進制加法沒檢查**溢出**。
- 忘記 **Big-5** 是繁體、**GB** 是簡體。
- 複製 `=AVERAGE(B2:B13)` 公式時沒用絕對引用。
