# 校本評核（SBA）總覽

> **權重：** 全科 20% · **學校考生必須完成** · 自修生豁免（成績 100% 基於公開考試）。

SBA 由教師指導的**兩項任務**組成，聚焦**資訊系統的設計、實現、測試與評估**。

## 分數分配

| 任務 | 重點 | 分數 |
|------|------|------|
| **任務一** | 構思與應用（Design & Implementation） | 25 |
| **任務二** | 測試與評估（Testing & Evaluation） | 15 |
| **合計** | | **40** |

40 分按比例換算到 20%。

## 實施方式

- 老師在中五收到 HKEAA 核准的**任務簡介**。
- 任務通常在**課堂**內完成，老師監督。
- 老師批改後提交至 HKEAA。
- HKEAA 可能跨校調節分數。
- 你**必須保留作品**至成績公佈。

## 主題選項

SBA 主題與你的**選修**掛鈎：

- **選 2A** → 資料庫系統（庫存、圖書館、預約）
- **選 2B** → Web 應用（小商店、投票、日程）
- **選 2C** → 桌面/裝置程序（小遊戲、傳感器記錄器）

多數學校把 2A + 2B 合併成一個 Web + 資料庫系統，兩項選修一併覆蓋。

## 推薦流程

```
中五上 ── 定題、做設計（ER 圖、原型）
中五下 ── 實現核心功能
中五暑假 ── 與同學進行可用性測試
中六上 ── 完善、定稿、提交
```

## 評分準則總結

雖然每項任務有自己的評分細則，HKEAA 通常關注：

1. 清晰**識別系統要解決的問題**。
2. **有文件的設計**（用例、ER 圖、線框圖、流程圖）。
3. **可執行的實現**符合設計 —— 部分實現可得部分分。
4. **測試證據** —— 測試用例、預期 vs 實際、截圖。
5. **反思評估** —— 哪裏好、哪裏不足、改進空間。
