# 歷屆試題策略

> 歷屆試題是中六最好的老師。用得有方法，不是湊數。

## 來源

- **HKEAA 官方試卷集**（P1 + P2 含評分參考）。
- **樣卷**（針對 2025+ 新課綱，價值極高）。
- **學校模擬卷** —— 通常基於官方題。

## 建議節奏

| 階段 | 頻率 | 量 |
|------|------|---|
| 中五下 | 每月 1 套 | 1 卷一 |
| 中五暑前 | 每 3 周 | 1 卷一 + 1 卷二 |
| 中六上 | 每週 | 1 套完整，計時 |
| 中六下 | 每週 2 次 | 定向部分 + 批改 |

## 正確做題流程

1. **計時坐定**。無暫停、無手機。
2. **只用考場會有的材料**（計算器 + 試卷內參考表）。
3. **用官方評分參考自評**。
4. **每個錯題標** 模組 + 子話題。
5. **一週後重做**漏掉的題。

## 跨年常見模式

- 卷一 MC 的**進制轉換**（幾乎年年）。
- 卷一 B 部分**偽程式碼追蹤**（年年）。
- 卷 2A 的**多表 JOIN**（年年）。
- 卷一的 **HTTP / DNS**（多數年）。
- 卷 2B 的 **PHP + 資料庫**（2026 起）。

## 批改後的分析

| Q# | 主題 | 得分 | 失分原因 | 行動 |
|----|------|------|---------|------|
| 1 | MC 二進制 | 0/1 | 漏讀 NOT | 否定 MC 放慢 |
| 4(b) | SQL JOIN | 2/4 | JOIN 錯表 | 練 3 表 JOIN |
| 6(c) | 偽程式碼 | 3/5 | 漏 END FOR | 用 HKEAA 語法 |

5 套以後，這張表就是你的弱點清單。

## 學生常丟的小分

- **沒看分值** → 太短/太長。
- **跳過結構題簡單子題**，因為第一部分太難。
- **忘單位**（KB vs MB、秒 vs 毫秒）。
- **不標圖**。
- **「兩」當「多」**（題問「列出兩個理由」，寫兩個）。

::: tip 重放歷屆 SQL
卷 2A 與 SBA 備考時，把歷屆試題的 schema 在 **[SQL Books](https://sqlbooks.codenav.dev)** 復刻成模板。每年一份沙盒。回到任意年份，改一行 clause 看輸出差異 —— 這種深度玩法，紙面溫習給不了。
:::
