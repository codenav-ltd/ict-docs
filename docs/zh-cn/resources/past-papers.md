# 历届试题策略

> 历届试题是中六最好的老师。用得有方法，不是凑数。

## 来源

- **HKEAA 官方试卷集**（P1 + P2 含评分参考）。
- **样卷**（针对 2025+ 新课纲，价值极高）。
- **学校模拟卷** —— 通常基于官方题。

## 建议节奏

| 阶段 | 频率 | 量 |
|------|------|---|
| 中五下 | 每月 1 套 | 1 卷一 |
| 中五暑前 | 每 3 周 | 1 卷一 + 1 卷二 |
| 中六上 | 每周 | 1 套完整，计时 |
| 中六下 | 每周 2 次 | 定向部分 + 批改 |

## 正确做题流程

1. **计时坐定**。无暂停、无手机。
2. **只用考场会有的材料**（计算器 + 试卷内参考表）。
3. **用官方评分参考自评**。
4. **每个错题标** 模块 + 子话题。
5. **一周后重做**漏掉的题。

## 跨年常见模式

- 卷一 MC 的**进制转换**（几乎年年）。
- 卷一 B 部分**伪代码追踪**（年年）。
- 卷 2A 的**多表 JOIN**（年年）。
- 卷一的 **HTTP / DNS**（多数年）。
- 卷 2B 的 **PHP + 数据库**（2026 起）。

## 批改后的分析

| Q# | 主题 | 得分 | 失分原因 | 行动 |
|----|------|------|---------|------|
| 1 | MC 二进制 | 0/1 | 漏读 NOT | 否定 MC 放慢 |
| 4(b) | SQL JOIN | 2/4 | JOIN 错表 | 练 3 表 JOIN |
| 6(c) | 伪代码 | 3/5 | 漏 END FOR | 用 HKEAA 语法 |

5 套以后，这张表就是你的弱点清单。

## 学生常丢的小分

- **没看分值** → 太短/太长。
- **跳过结构题简单子题**，因为第一部分太难。
- **忘单位**（KB vs MB、秒 vs 毫秒）。
- **不标图**。
- **「两」当「多」**（题问「列出两个理由」，写两个）。

::: tip 重放历届 SQL
卷 2A 与 SBA 备考时，把历届试题的 schema 在 **[SQL Books](https://sqlbooks.codenav.dev)** 复刻成模板。每年一份沙盒。回到任意年份，改一行 clause 看输出差异 —— 这种深度玩法，纸面复习给不了。
:::
