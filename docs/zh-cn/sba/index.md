# 校本评核（SBA）总览

> **权重：** 全科 20% · **学校考生必须完成** · 自修生豁免（成绩 100% 基于公开考试）。

SBA 由教师指导的**两项任务**组成，聚焦**信息系统的设计、实现、测试与评估**。

## 分数分配

| 任务 | 重点 | 分数 |
|------|------|------|
| **任务一** | 构思与应用（Design & Implementation） | 25 |
| **任务二** | 测试与评估（Testing & Evaluation） | 15 |
| **合计** | | **40** |

40 分按比例换算到 20%。

## 实施方式

- 老师在中五收到 HKEAA 核准的**任务简介**。
- 任务通常在**课堂**内完成，老师监督。
- 老师批改后提交至 HKEAA。
- HKEAA 可能跨校调节分数。
- 你**必须保留作品**至成绩公布。

## 主题选项

SBA 主题与你的**选修**挂钩：

- **选 2A** → 数据库系统（库存、图书馆、预约）
- **选 2B** → Web 应用（小商店、投票、日程）
- **选 2C** → 桌面/装置程序（小游戏、传感器记录器）

多数学校把 2A + 2B 合并成一个 Web + 数据库系统，两项选修一并覆盖。

## 推荐流程

```
中五上 ── 定题、做设计（ER 图、原型）
中五下 ── 实现核心功能
中五暑假 ── 与同学进行可用性测试
中六上 ── 完善、定稿、提交
```

## 评分准则总结

虽然每项任务有自己的评分细则，HKEAA 通常关注：

1. 清晰**识别系统要解决的问题**。
2. **有文档的设计**（用例、ER 图、线框图、流程图）。
3. **可运行的实现**符合设计 —— 部分实现可得部分分。
4. **测试证据** —— 测试用例、预期 vs 实际、截图。
5. **反思评估** —— 哪里好、哪里不足、改进空间。
