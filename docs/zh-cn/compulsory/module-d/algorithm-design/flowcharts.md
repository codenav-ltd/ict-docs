# 2.2 · 流程图

> **目标：** 用标准符号读画流程图。

## 标准符号

| 符号 | 形状 | 用途 |
|--------|-------|-----|
| **Terminator 终止** | 圆角矩形 / 椭圆 | Start、End |
| **Process 过程** | 矩形 | 动作（赋值、计算） |
| **Input / Output 输入 / 输出** | 平行四边形 | 读 / 写 |
| **Decision 判断** | 菱形 | 是 / 否问题 |
| **Flow line 流向** | 箭头 | 执行方向 |
| **Connector 连接器** | 小圆 | 接到别处（避线交） |
| **Subroutine call 子程序调用** | 带双竖线的矩形 | 调用另一子程序 |

## 例 · BMI 类别流程图

```
            ┌────────────┐
            │  Start     │
            └────┬───────┘
                 │
            ┌────▼────────────┐
            │ Input weight, h │
            └────┬────────────┘
                 │
            ┌────▼─────────┐
            │ bmi ← w/h²   │
            └────┬─────────┘
                 │
              ◇──┴──◇  YES   ┌──────────────┐
              ◇bmi<18.5◇──▶  │Output "Under"│
              ◇──┬──◇         └────┬─────────┘
                 │ NO               │
              ◇──┴──◇  YES   ┌──────────────┐
              ◇bmi<24◇──▶    │Output "Norm" │
              ◇──┬──◇         └────┬─────────┘
                 │ NO               │
              ┌──┴──────────────┐   │
              │Output "Over/Obese"│ │
              └────┬─────────────┘ │
                   └──────┬────────┘
                          ▼
                     ┌────────┐
                     │   End  │
                     └────────┘
```

（HKEAA 以其漂亮的印刷形式重现流程图 —— 但符号集相同。）

## 好流程图的规则

- 每个过程**一入一出**。
- **流向线**不应交叉（用连接器）。
- **判断菱形**恰好一个 YES、一个 NO 分支。
- 嵌套不过 3 层 —— 更深就分解为子流程图。

## 何时用流程图 vs 伪代码

| 用流程图 | 用伪代码 |
|---------------------|---------------------|
| 观众偏好视觉 | 观众能读代码风格 |
| 展示分支逻辑 | 展示详细步骤序列 |
| SBA 设计文档 | 编码实现笔记 |

## 考试式题目

> **题（5 分）：** 画流程图：读学生分数，≥ 50 输出「Pass」，否则「Fail」。

**参考答案（文字表示）：**

1. Start。
2. Input mark。
3. 判断：mark ≥ 50？
   - YES → Output 「Pass」 → End。
   - NO → Output 「Fail」 → End。

考试时实际画图形 —— 终止、平行四边形给输入、菱形给判断、平行四边形给输出、终止 End。

## 关键要点

- 五种符号几乎涵盖一切。
- 一入 / 一出；不交叉。
- 流程图配伪代码更清楚。
