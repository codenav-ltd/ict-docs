## a. 基本机器组织（14h）

### 学习要点

- 解释硬件功能：**输入/输出装置、处理单元、总线、存储装置**。
- 解释 **CPU** 结构与组成部分。
- 说明 **取指 - 译码 - 执行** 周期及寄存器、总线、ALU 的角色。
- 描述 **RAM、ROM、Cache** —— 以及内存大小、地址、字长与性能的关系。
- 描述输入输出装置并按情境选用。
- 描述存储装置：存取方式、易失性、传输速率、容量。
- 说明处理器、内存、二级存储、数据通信的**最新发展**。

### CPU 组件

```
          ┌────────────── CPU ───────────────┐
 输入  ▶ │  ┌─────────┐  ┌────┐  ┌────────┐ │ ▶ 输出
         │  │ 控制单元│  │ALU │  │寄存器  │ │
         │  └─────────┘  └────┘  │PC IR等 │ │
         │       ▲ 总线系统 ▲     └────────┘ │
         └───────│─────────│─────────────────┘
                 ▼         ▼
          ┌──────────┐   ┌────────────┐
          │  内存    │   │ 存储装置  │
          │ (RAM/ROM)│   │ (HDD/SSD)  │
          └──────────┘   └────────────┘
```

### 取指 - 译码 - 执行周期

1. **取指（Fetch）** —— PC 告诉 MAR 要读取的地址，数据返回 MDR。
2. **译码（Decode）** —— 控制单元解码 IR 中的指令。
3. **执行（Execute）** —— ALU 执行算术/逻辑；结果放入寄存器或内存。
4. **递增** PC，重复。

### CPU 速度单位

| 单位 | 时间 | 用途 |
|------|------|------|
| 1 毫秒（ms） | 10⁻³ 秒 | 磁盘寻道时间 |
| 1 微秒（μs） | 10⁻⁶ 秒 | 旧 RAM 周期 |
| 1 纳秒（ns） | 10⁻⁹ 秒 | 现代 RAM 存取 |
| 1 皮秒（ps） | 10⁻¹² 秒 | CPU 指令时间 |

### 内存类型

| 类型 | 易失？ | 速度 | 价格 | 用途 |
|------|--------|------|------|------|
| **Cache** | 是 | 最快 | 最贵 | 紧贴 CPU 的活跃数据 |
| **RAM（DRAM）** | 是 | 快 | 中等 | 程序运行时的主内存 |
| **ROM** | 否 | 快（只读） | 中等 | 固件（如 BIOS） |
| **二级存储**（HDD/SSD/Flash） | 否 | 最慢 | 便宜 | 长期储存 |

### 容量单位前缀

```
1 KB = 1024 字节（不是 1000！）
1 MB = 1024 KB
1 GB = 1024 MB
1 TB = 1024 GB
```

SI 用 10 的幂，所以厂商宣传的 1 TB（10¹² 字节）比真正的 1 TB（2⁴⁰ 字节）**小**。你应能解释为什么 Windows 显示的容量比包装少。

### 输入输出装置

| 装置 | 类型 | 用途 |
|------|------|------|
| 键盘、鼠标、扫描仪、麦克风、摄像头、传感器 | 输入 | 捕获用户动作或环境 |
| 显示器、扬声器、打印机、绘图仪、投影仪 | 输出 | 显示或产出结果 |
| 触摸屏 | 兼输入输出 | 移动 UI |

### 存储装置

| 装置 | 存取 | 易失？ | 大致传输速率 | 容量 |
|------|------|--------|--------------|------|
| 磁盘（HDD） | 随机 | 否 | ~100 MB/s | 最高 ~20 TB |
| 光盘（CD/DVD/Blu-ray） | 随机 | 否 | ~50 MB/s | 最高 ~100 GB |
| 闪存（SSD / USB） | 随机 | 否 | ~500 MB/s (SATA), ~7 GB/s (NVMe) | 最高 ~8 TB |
| 磁带 | 顺序 | 否 | ~400 MB/s | ~20 TB+ |
| 网络存储（NAS / 云） | 网络随机 | 否 | 取决于网络 | 几乎无限 |

趋势：体积更小、容量更大、速率更快。

## b. 系统软件（6h）

### 学习要点

- 了解**系统软件与应用软件**的功能与硬件、用户之间的关系。
- 概述**操作系统**的基本功能及常见 OS。
- 说明**实用程序**与**驱动程序**的功能与必要性。
- 区分各种**运行模式**（批处理、实时、并行、分布式、虚拟化）。

### 软件层次

```
┌────────────────────────────┐
│   应用软件                  │  Word、Chrome、Photoshop…
├────────────────────────────┤
│   系统软件                  │  OS + 实用程序 + 驱动
├────────────────────────────┤
│   硬件                      │  CPU、RAM、装置…
└────────────────────────────┘
       ▲
       │ 用户主要通过应用软件交互。
```

### 操作系统功能

- **进程管理** —— 调度与切换。
- **内存管理** —— 分配 RAM、虚拟内存分页。
- **文件管理** —— 目录、权限。
- **设备管理** —— 驱动与 I/O。
- **安全与用户管理** —— 登录、权限。
- **用户界面** —— GUI 或 CLI。

### 常见 OS

| OS | 典型用途 |
|----|---------|
| Windows | 桌面、办公 |
| macOS | 创意专业 |
| Linux（Ubuntu、Debian、Fedora） | 服务器、开发者 |
| iOS、Android | 移动设备 |
| Chrome OS | 轻量笔记本 |

### 实用程序

| 实用程序 | 功能 |
|---------|------|
| 数据压缩 | 减小文件体积（zip、rar、7z） |
| 病毒检测 | 检测并清除恶意软件 |
| 文件管理器 | 浏览、复制、移动、重命名 |
| 磁盘碎片整理 | 重组 HDD 上的数据以加速访问 |
| 系统监控 | 显示 CPU/RAM/磁盘使用 |

### 驱动程序

**驱动程序**让 OS 与特定硬件通信。没有正确驱动，设备即使物理连接也无法使用。

### 运行模式

| 模式 | 含义 | 例子 |
|------|------|------|
| **批处理** | 任务排队、无人交互运行 | 月薪发放 |
| **实时处理** | 需立即响应 | 飞控、网银 |
| **并行处理** | 多处理器同时处理一个任务的不同部分 | 科学模拟 |
| **分布式处理** | 工作分散在多台联网计算机 | 云服务 |
| **虚拟化** | 一台物理机运行多个虚拟机 | 云服务器、沙盒 |

## 常见错误

- 把 **cache**（易失）当成 **ROM**（非易失）。
- 混淆 **固件**（储存在 ROM 的软件）与操作系统。
- 把 Windows 既算系统软件又算应用软件（它是系统软件）。
- 忘记 **虚拟化** 是一种运行模式，而不仅指 VirtualBox。

## 历届考题热点

- MC：判断哪种内存是易失/非易失。
- MC：在（成本、速度、便携性）权衡中选择最佳存储装置。
- Section B：解释取指-译码-执行周期及总线的角色。
