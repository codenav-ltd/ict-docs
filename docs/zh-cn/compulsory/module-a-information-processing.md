## a. 资讯处理简介（3h）

### 学习要点

- 识别并分析**信息系统**的组成。
- 区分各种**信息处理过程**。
- 说出**数据与信息**的区别，并识别文字、数字、图像、音频、视频等数据类型。
- 定义**信息时代**，讨论**信息素养**在知识型社会中的重要性。

### 概念

- **信息系统的组成：** 目的、数据、过程、技术、人员。
- **信息处理过程：** 收集、组织、分析、储存、处理、传输、展示。
- **数据 vs 信息：** 数据是原始材料；信息是被赋予上下文与意义的数据。

### 常见题目句式

- *「列出数据与信息的两点差异。」*
- *「找出上述情境中的输入 / 处理 / 输出组件。」*

## b. 数据组织与控制（4h）

### 学习要点

- 在层级化数据组织中识别**数据、记录、字段、文件、数据库**。
- 解释记录如何被**组织、储存、检索**。
- 陈述**直接存取**与**顺序存取**的优缺点和应用。
- 讨论**数据控制**的需要并描述错误检测方法。

### 关键知识

| 术语 | 定义 |
|------|------|
| 字段（Field） | 最小的有意义数据单元（如学生的姓） |
| 记录（Record） | 相关字段的集合（如一个学生的整行数据） |
| 文件（File） | 相关记录的集合 |
| 数据库（Database） | 相关文件的结构化集合 |
| 直接（随机）存取 | 用地址直接跳到某条记录。个别查询快，硬盘常用。 |
| 顺序存取 | 从头按顺序读。磁带成本低，但随机读慢。 |

### 错误侦测 vs 防止

| 技术 | 目的 |
|------|------|
| **验证（Validation）** | 软件自动检查（范围、必填、格式、校验码） |
| **核实（Verification）** | 人工检查（重新输入、目视检查、双人输入） |
| **奇偶校验（Parity Check）** | 位级别检查（奇/偶校验位）用于传输和储存 |

::: tip 记忆口诀
「**V**alidation 由代码 **V**alidate，**V**erification 由人 **V**iew。」
:::

## c. 数据表示（10h）

### 学习要点

- 区分**模拟数据**与**数字数据**。
- 在**十进制 / 二进制 / 十六进制**之间互转。
- 进行简单的**二进制加减**，分析**溢出**。
- 用**二进制补码**表示负整数。
- 知道字符如何用 **ASCII、Big-5、GB、Unicode** 表示。
- 简单了解**图像、音频、视频**的数字化方式。
- 比较常见**文件格式**（bmp、png、jpg、wav、mp3、avi、mpeg4、txt、docx、odt、pdf）。

### 进制对照表

```text
十进制  二进制    十六进制
0       0000      0
1       0001      1
2       0010      2
...
9       1001      9
10      1010      A
11      1011      B
12      1100      C
13      1101      D
14      1110      E
15      1111      F
```

### 二进制补码（8 位示例）

用 8 位表示 **−5**：

1. 写出 **+5** 的二进制：`0000 0101`
2. 取反：                  `1111 1010`
3. 加 1：                  `1111 1011`

### 字符编码

| 编码 | 大小 | 用途 |
|------|------|------|
| ASCII | 7 位（储存为 1 字节） | 基础英文 |
| Big-5 | 2 字节 | 繁体中文 |
| GB | 2–4 字节 | 简体中文 |
| Unicode（UTF-8） | 1–4 字节 | 全球所有文字 |

> 课程指引说**无需**背诵具体编码，但要理解为什么更大字符集需要更多位数。

### 文件格式比较

| 类型 | 常见格式 | 压缩 | 适合 |
|------|----------|------|------|
| 图像 | bmp、png、jpg | bmp 不压缩，png 无损，jpg 有损 | jpg 用于照片，png 用于图标/截图 |
| 音频 | wav、mp3 | wav 不压缩，mp3 有损 | mp3 用于流媒体/分发 |
| 视频 | avi、mpeg4 | 视情况，mpeg4 有损 | mpeg4 用于网络 |
| 文档 | txt、docx、odt、pdf | 视情况 | pdf 归档、txt 纯文本 |

## d. 数据操控与分析（20h）

最大的子话题，考得最多。分为电子表格与数据库两部分。

### 电子表格技能

- **单元格引用：** 相对（`A1`）、绝对（`$A$1`）、混合（`$A1`、`A$1`）
- **函数：** SUM、AVERAGE、MIN、MAX、COUNT、COUNTA、COUNTIF、IF、IFS、VLOOKUP、HLOOKUP、INDEX、MATCH、ROUND、INT、MOD、ABS
- **数学、逻辑、关系运算符**
- 单条件或多条件的**筛选、查找、排序**
- **跨工作表**动态引用
- **数据透视表与透视图**做聚合
- **What-if 分析** —— 单变量求解、方案管理器、模拟运算表

### 数据库技能（DBMS）

- 应用数据组织概念创建并维护**简单数据库**。
- 创建并使用**表单**录入数据。
- 通过**查询**提取与操控数据，创建**报表**。
- 追踪并理解简单的 **SQL 语句**（不要求写复杂 SQL，那是卷二 2A 的内容）。

::: tip 不必为读懂 SELECT 就装 MySQL
在模块 A 阶段你只需要看懂 `SELECT … FROM … WHERE`。打开 **[SQL Books 在线编辑器](https://sqlbooks.codenav.dev)**，粘贴示例，点击运行 —— 立刻看到结果。不用装 XAMPP，不会有版本冲突。
:::

### 示例 · 电子表格

> **题：** 工作表 `B2:B13` 储存 12 个月销售额。在 `C2` 写一个公式：若 `B2` 大于年度平均值显示「Above」，否则显示「Below」。

```text
=IF(B2 > AVERAGE($B$2:$B$13), "Above", "Below")
```

注意复制公式时需用 `$` 锁定范围。

### 示例 · SQL 追踪

```sql
SELECT name, score
FROM   students
WHERE  class = 'F4A'
ORDER  BY score DESC;
```

预期：返回 F4A 班所有学生的 `name` 和 `score`，按 `score` 降序。

## 常见错误

- 混淆 **validation** 与 **verification**（用上面的 V 口诀记忆）。
- 二进制加法没检查**溢出**。
- 忘记 **Big-5** 是繁体、**GB** 是简体。
- 复制 `=AVERAGE(B2:B13)` 公式时没用绝对引用。
