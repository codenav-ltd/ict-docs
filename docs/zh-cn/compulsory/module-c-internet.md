## a. 网络与互联网基础（9h）

### LAN vs WAN

| 特性 | LAN | WAN |
|------|-----|-----|
| 覆盖 | 单个建筑/校园 | 城市、国家、全球 |
| 拥有者 | 通常单一机构 | 多方 |
| 速度 | 较高（1 Gbps+） | 单用户较低 |
| 例子 | 学校机房 | 互联网本身 |

### IPv4 vs IPv6

| 特性 | IPv4 | IPv6 |
|------|------|------|
| 长度 | 32 位 | 128 位 |
| 表示 | 点分十进制：`192.168.1.10` | 冒号十六进制：`2001:db8::1` |
| 地址空间 | ~43 亿 | ~3.4 × 10³⁸ |
| 为何需要 IPv6 | IPv4 地址用尽 | 面向未来、路由简化 |

### 网络硬件

| 装置 | 功能 |
|------|------|
| **NIC** | 把计算机接入网络 |
| **交换机** | 用 MAC 地址在 LAN 内转发帧 |
| **路由器** | 用 IP 地址在网络间转发数据包 |
| **调制解调器** | ADSL/电缆/光纤的调制解调 |
| **无线接入点** | 让无线客户端接入有线 LAN |

### 传输介质

- **UTP**（非屏蔽双绞线）—— 铜线，办公室常见
- **同轴电缆** —— 旧式、有线电视
- **光纤** —— 光在玻璃中，速度最快、距离最远
- **微波 / 卫星** —— 长距离无线
- **蓝牙 / Wi-Fi** —— 短距离无线

### 互联网接入比较

| 方式 | 速度 | 价格 | 移动性 | 备注 |
|------|------|------|--------|------|
| 拨号 | ~56 kbps | 便宜 | 无 | 已淘汰 |
| ADSL | 最高 ~24 Mbps | 低 | 无 | 用电话线 |
| 有线 | 最高 ~1 Gbps | 中 | 无 | 用电视线 |
| 光纤入户 | 1–10 Gbps | 中 | 无 | 香港主流 |
| 4G LTE | ~100 Mbps | 中 | 高 | 移动 |
| 5G | 1–10 Gbps | 高 | 高 | 最新移动 |
| 卫星 | 不定 | 高 | 高 | 偏远地区 |

### TCP/IP 基础

- **TCP** 提供可靠、顺序、错误检查的传输（按**端口号**区分服务）。
- **IP** 负责**寻址**与**路由**。
- 二者合作实现端到端通信。

## b. 互联网服务与应用（5h）

### URL 结构

```
https:// www.example.com :443 /path/to/page ?id=123 #section
└──┬──┘ └──────┬───────┘ └┬─┘ └────┬─────┘ └──┬──┘ └──┬───┘
 scheme    hostname     port      path     query  fragment
```

### DNS 示意

```
用户输入  www.example.com
       │
       ▼
   ┌────────────┐    ┌────────────┐
   │  浏览器    │ ─▶ │ DNS 服务器 │  解析 www.example.com
   └────────────┘    └────────────┘
        ▲                  │
        │ IP 93.184.216.34 │
        └──────────────────┘
        │
        ▼
   向 93.184.216.34 发 HTTPS 请求
```

### HTTP vs HTTPS

| 特性 | HTTP | HTTPS |
|------|------|-------|
| 加密 | 无 | TLS/SSL |
| 默认端口 | 80 | 443 |
| 隐私 | 明文 | 加密 |
| 现今使用 | 很少 | 几乎全部 |

### Web 文件格式

| 需要 | 格式 |
|------|------|
| 图像 | jpg/jpeg、png、gif、webp、svg |
| 音频 | mp3、ogg、wav |
| 视频 | mp4（h.264/h.265）、webm |
| 文档 | pdf |

### 互联网服务

| 服务 | 协议 | 用途 |
|------|------|------|
| Web | HTTP/HTTPS | 浏览 |
| Email | SMTP、POP3、IMAP | 收发邮件 |
| FTP | FTP/SFTP | 文件传输 |
| 远程登录 | SSH | 服务器管理 |
| 聊天 | XMPP、专有 | 即时通信 |
| 流媒体 | RTMP、HLS、DASH | 语音、视频会议、直播 |

## c. 网页编写基础（3h）

### 最小 HTML 页面

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>你好 HKDSE</title>
</head>
<body>
  <h1>你好，世界</h1>
  <p>这是我的第一个网页。</p>
  <a href="https://www.hkeaa.edu.hk">HKEAA</a>
  <img src="logo.png" alt="logo">
</body>
</html>
```

不需要背诵所有标签 —— 但要认得结构。

### 网页组织原则

- **导航** 清晰
- **链接、表格、框架、多媒体** 放置得当
- 合理的**配色、背景、字体**
- 匹配**目标受众**（小朋友 vs 公司 vs 学术）

## d. 互联网的威胁与安全（14h）

### 威胁清单

| 威胁 | 做了什么 |
|------|---------|
| 病毒 | 附着在其他程序上自我复制 |
| 蠕虫 | 在网络上独立自我复制 |
| 木马 | 伪装成合法软件，开后门 |
| 间谍软件 | 未经同意追踪用户 |
| 勒索软件 | 加密数据索要赎金 |
| 未授权访问 | 无权限登入 |
| 窃听 | 捕获传输中的数据 |
| 通过动态网页入侵 | SQL 注入、XSS |
| DoS / DDoS | 用流量淹没服务 |

### 防御措施

| 措施 | 阻止 |
|------|------|
| 浏览器安全设置 | 钓鱼、悄悄下载 |
| 防病毒软件 | 设备恶意软件 |
| 认证（密码、2FA、生物） | 未授权访问 |
| 访问权限控制 | 内部滥用 |
| 防火墙 | 不期望的进出流量 |
| WPA / WPA2 / WPA3 | 无线窃听 |
| VPN | 加密用户与网络间数据 |
| 定期 OS / 软件更新 | 已知漏洞 |

### 加密基础

| 概念 | 备注 |
|------|------|
| 对称加密 | 一把共享密钥（如 AES） |
| 非对称加密 | 公钥/私钥对（如 RSA） |
| 香港 PKI | 香港邮政运营的公钥基础设施 |
| 密钥越大 | 越安全但越慢 |

### 认证与授权

- **认证** 回答「你是谁？」—— 密码、令牌、生物、数字证书。
- **授权** 回答「你能做什么？」—— 访问控制列表、角色。

### 在线交易安全

- **SSL/TLS** 把 HTTP 加密成 HTTPS。
- **智能卡、安全令牌** —— 硬件认证。
- **手机 SMS 一次性密码** —— 香港银行常用。

## 常见错误

- 把**互联网**称为 LAN（互联网是 WAN 的 WAN）。
- 混淆 **交换机**（只在 LAN）与 **路由器**（跨网络）。
- 以为 HTTPS 「完全安全」—— 它只保护传输。
