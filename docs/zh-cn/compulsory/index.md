# 卷一 · 必修部分总览

> **比重：** 全科 55% · **时长：** 2 小时 · **形式：** Section A（MC，22%）+ Section B（短/结构题，33%）

必修部分在 **卷一（Paper 1）** 考核。不论你的选修组合如何，每位考生都要做同一份卷一。因此它是 ICT 科目中分值最大的单一部分，也是你复习的首要目标。

## 五大模块速览

| # | 模块 | 课时 | 为什么重要 |
|---|------|------|----------|
| **A** | [资讯处理](./module-a-information-processing) | 37 | 卷二 2A 的基础。培养数据思维与 Excel / DBMS 技能。 |
| **B** | [电脑系统基础](./module-b-computer-system) | 20 | 硬件 / 软件能力。MC 题的稳定来源。 |

### 考核范围

模块 A–E 的所有学习要点都可能出现。HKEAA 经常在一道结构题里跨模块出题 —— 同一长题的不同子题可能从二进制（模块 A）跳到网络协议（模块 C），最后落到伪代码追踪（模块 D）。

### 卷一的评核目标

你需要展示：

1. **知识与理解** 电脑系统、数据、硬件、软件、互联网。
2. **应用** 把陈述性知识用到具体场景（如选择合适的验证方法）。
3. **分析** 比较两种解决方案、追踪伪代码、调试。
4. **综合** 用多个概念设计新的解决方案。

课程指引中的行为动词对应不同的认知层次：

| 动词类 | 认知层次 |
|--------|--------|
| state, list, define, recognise | 回忆 |
| describe, explain, identify, demonstrate, apply | 应用 |
| distinguish, analyse, compare, evaluate | 分析 |
| develop, plan, design, construct | 综合 |

题目以「**设计** 一个函数 …」开头，意味着要写完整的实现逻辑。以「**陈述** 两者的差别」开头，一句话即可。

## 必修部分学习建议

1. **顺序很重要。** 大部分老师按 A → C → B → E → D 教学，让前面的概念支撑后面的学习。自学者也建议如此。
2. **把模块 D 当成编程肌肉。** 编程不能临时抱佛脚。从中四起每周至少跑一个 Python（或 C++）程序。
3. **用间隔记忆背词汇。** 模块 B、C、E 词汇量大。闪卡比重读笔记更有效。
4. **与选修对照学习。** 模块 A 对应卷二 2A，模块 C 对应 2B，模块 D 对应 2C —— 一起复习省时间。

::: tip 在浏览器里练习 SQL
卷一 Section B 经常出 SQL `SELECT` 的输出预测题。在 **[SQL Books](https://sqlbooks.codenav.dev)** 上交互式跑这些例子 —— 免费的浏览器 SQL 工作台，每个账号自带 MySQL 沙盒，无需配置、不怕弄坏。
:::
