# 卷一策略

> 2 小时 · MC 22% + Section B 33% · 仅必修部分。

## 时间分配

| 部分 | 大致分数 | 建议时间 |
|------|---------|---------|
| Section A（MC） | 36 × 1 = 36 | 30–40 分钟 |
| Section B（短题 + 结构题） | ~55 | 70–80 分钟 |
| 检查 | — | 10 分钟 |

> 若 20 分钟做完 A，**别立刻进 B** —— 用 10 分钟复查 MC。MC 错最便宜挽回。

## Section A · 选择题

### 每题流程

1. **遮住选项**，先在脑里给出答案。
2. **打开选项** 选最接近的。
3. **不确定就排除** 至少 2 个错的。
4. **果断推进**，别一题纠缠 3 分钟。
5. **标记不确定** 的，留时间回看。

### MC 常见陷阱

| 陷阱 | 例子 |
|------|------|
| 几乎正确 | 「TCP 做什么？」A) 路由包（那是 IP）B) 可靠传输（正确） |
| 否定式 | 「以下哪项**不是** normalisation 的好处？」漏看 NOT |
| 单位不符 | 「1 KB =」 1024 字节（不是 1000） |
| 范围混淆 | 题描述 LAN，但选项对比 WAN |

## Section B · 短题与结构题

### 结构题结构

通常 4–6 个子题串成一道：

```
Q4. 情境段：学校图书馆系统
   (a) 定义 X。 [2]
   (b) 识别 Y 的两个属性。 [2]
   (c) 写 Z 的伪代码。 [4]
   (d) 评估设计相对 W 的优劣。 [4]
```

每个子题当作小论文。情境段只在引用时重读。

### 答案长度按分数

| 分数 | 典型长度 |
|------|---------|
| 1 | 一短语或单一事实 |
| 2 | 两事实 或 一事实 + 一例 |
| 3–4 | 含定义 + 例的结构段 |
| 5–6 | 多段 + 图 + 比较 |

### 命令动词认真对待

| 动词 | 期望回答 |
|------|---------|
| **State / List** | 简短事实 |
| **Describe** | 句级解释 |
| **Explain** | 为什么/如何，含因果 |
| **Compare** | 两侧表或段 |
| **Evaluate** | 优 + 劣 + 结论 |
| **Design** | 计划 + 理由（如 ER 图 + 解释） |

### 答题模板

#### 「解释为何 X 发生」

> X 发生因为 **\<原因\>**。这是因为 **\<机制\>**。结果是 **\<效果\>**。例如 **\<具体实例\>**。

#### 「比较 A 与 B」

> A 是 **\<简述\>**，而 B 是 **\<简述\>**。就 **\<标准 1\>** 而言，A **\<…\>**，B **\<…\>**。就 **\<标准 2\>** 而言，A **\<…\>**，B **\<…\>**。因此 A 更适合 **\<场景\>**。

#### 伪代码作答

务必 HKEAA 风格：

```text
INPUT n
total ← 0
FOR i ← 1 TO n
    total ← total + i
END FOR
OUTPUT total
```

用 `←` 不用 `=`。用 `END FOR`、`END IF` 收尾。
