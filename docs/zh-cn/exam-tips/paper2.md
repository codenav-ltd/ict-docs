# 卷二策略

> 1 小时 30 分钟 · 全科 25% · **仅答两个选修**。

## 时间分配

| 部分 | 建议时间 |
|------|---------|
| 翻卷找到你的两选修 | 5 分钟 |
| 选修 1 | 40 分钟 |
| 选修 2 | 40 分钟 |
| 检查 | 5 分钟 |

> 最大浪费：先做弱的选修。**先做你强的** —— 攒分增信心。

## 各选修策略

### 卷 2A · 数据库

- **参考表** 知道在哪。不背 SQL 关键字 —— 直接看参考。
- **写 SQL 顺序：**
  1. `FROM` / `JOIN`（涉及哪些表？）
  2. `WHERE`（过滤行）
  3. `GROUP BY`
  4. `HAVING`
  5. `SELECT`
  6. `ORDER BY`
  7. `LIMIT`
- **JOIN 时给表起别名**（`Student s INNER JOIN Score sc`）。
- 设计题旁**先画 ER 图**。
- 语句末尾加 `;`，显得专业。

### 卷 2B · 网络应用开发

- 明确区分**客户端 vs 服务器端**职责。
- 用户输入**绝不信任**，客户端 + 服务器端**双重校验**。
- PHP 数据库代码用**预处理**。
- HTML 包含 `<!DOCTYPE html>` 和 `<meta charset="UTF-8">`。
- CSS 题展示至少一个选择器、属性、值。

### 卷 2C · 算法与编程

- 题目未明示用 Python，**伪代码优先**。
- 复杂代码加**注释**。
- **跑一遍样例**再宣告正确。
- 准备**比较算法**（O 记号好用即使非必要）。

## 评分方式

评卷用「踩点」式：

- **一个概念 = 一分**。
- 即使最终答错有**方法分**（如 JOIN 结构正确但出现笔误）。
- 清晰加**额外分**（注释、缩进、起名）。

> 不确定时，写**更结构化、更清晰**的内容，未必更长。

## 读题清单

1. 圈**命令动词**（Write、Explain、Identify…）。
2. 圈**数据上下文**（哪张表？哪种语言？）。
3. 看**分值**。
4. 找**约束**（「用 JOIN」、「不用子查询」、「只一个循环」）。

## 卷二专属常见错误

- **2A：** 用 `WHERE COUNT(*) > 5`（应 `HAVING`）。
- **2A：** 把 `<>` 写成 `!=`。
- **2B：** 题问 cookie 却用了 localStorage。
- **2B：** 不用 `PDO::prepare()` → 可被 SQL 注入。
- **2C：** `for i in range(n)` 与 `range(1, n+1)` 弄错。
